# Chainlit UI Implementation Plan

## 現状の課題
- `chainlit_app.py`:
  - `progress_callback` が呼び出されるたびに `cl.Message` を新規送信しており、チャット欄が溢れて見通しが悪い。
  - レポート生成後の視覚的要約 (JSON) のパースが不安定で、失敗するとエラーで止まる可能性がある。
  - `astream` のイベントループ内でのメッセージ更新が不完全で、特定のノードの結果が表示されないことがある。

- `chainlit_app_v2.py`:
  - `cl.Step` を使用しており UI は整理されているが、`progress_callback` が `config_dict` に渡されていないため、非対話モード (ResearchLoop) の詳細な進捗が見えない。
  - 最終レポートの表示方法が未完成。

- `core/graph.py`:
  - 非対話モード時に `ResearchLoop` を呼び出し、グラフ外で並列実行するため、UI 側での制御が難しい。

## 実装方針
1. **統合された UI クラスの導入 (または既存のリファクタリング)**:
   - 進捗状況を一元管理する `ResearchProgressManager` のような仕組みを考え、Chainlit のメッセージを動的に更新する。
   - `cl.Step` を積極的に使い、階層的な進捗表示を実現する。

2. **堅牢な視覚的要約**:
   - JSON 抽出ロジックを強化し、不完全な JSON でも可能な限り修復して表示する。
   - `pyvis` で生成したネットワーク図をサイドバーまたはインラインで適切に表示する。

3. **非対話モードと対話モードの統一的な扱い**:
   - `ResearchLoop` からのコールバックを `cl.Step` でラップし、並列実行される各セクションの進捗を個別のステップとして表示する。

4. **最終レポートの改善**:
   - 長大なマークダウンを適切に表示（必要に応じてファイル添付へ誘導）。
   - 引用文献 (Sources) を見やすく表示。

## 最近の改善内容
- **フロントエンド安定性 (startsWith エラーの完全回避)**: 
  - Chainlit 2.10.x の `cl.File` に起因する JS クラッシュ (`startsWith` null error) を回避するため、ファイルの添付を一時的に廃止。
  - 代替案として `cl.Action`（アクションボタン）による **「クリップボードへのコピー機能」** を実装。
- **スレッドセーフなレポート取得**: 
  - `cl.user_session` にレポート全文を保持し、ボタンクリック時に Markdown コードブロックとして送信。Chainlit 標準のコピー機能を利用可能に。
- **進捗表示の同期**: 順次実行（対話モード）時のクエリ表示不一致を、`progress_callback` の伝搬により解消。
