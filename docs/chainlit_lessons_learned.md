# Chainlit UI 開発の知見と課題

## 成果
- **階層的な進捗表示**: `cl.Step` を活用し、リサーチの各工程（計画、検索、分析）を整理して表示するUXを構築した。
- **並列処理の可視化**: `UIProgressManager` により、複数のセクションが同時に進行するリサーチの進捗をユーザーに分かりやすく伝える仕組みを実装した。
- **対話モードの統合**: プランの承認や修正をチャットフローの中で自然に行えるようにした。

## 課題と致命的な不具合
1. **ファイル提供機能の不安定さ (`startsWith` エラー)**:
   - `cl.File` 要素を使用してファイルをダウンロードさせようとすると、フロントエンドで `Cannot read properties of null (reading 'startsWith')` という JS エラーが発生し、UI がクラッシュすることが頻発した。
   - 原因は、フロントエンドに渡される要素オブジェクトの `path` や `url` が `null` になる Chainlit 本体のバグ（あるいは特定の MIME タイプへの過敏な反応）である。
2. **静的ファイル配信の制約**:
   - `public/` ディレクトリを用いた直接リンク方式も試みたが、Chainlit の SPA ルーターがリンクを横取りしてしまい、ダウンロードではなく画面遷移（404）として処理されるなど、確実なファイル提供が困難であった。
3. **MIME タイプの厳格性**:
   - HTML や特定のアプリケーション形式のファイルを送る際、フロントエンド側で厳格なチェック（あるいは誤判定）が走り、警告やクラッシュを招くことがあった。

## Streamlit への移行に向けた知見
- **確実なダウンロード**: Streamlit の `st.download_button` はブラウザの標準的な動作に基づいているため、クラッシュのリスクが極めて低い。
- **柔軟なコンポーネント配置**: チャット形式に縛られず、サイドバーや展開可能なパネル（expander）を活用して、進捗表示と結果表示を分離できる。
- **状態管理の簡素化**: `st.session_state` により、複雑なリサーチ状態（グラフのステート、取得したソース等）をセッションごとに容易に管理できる。
